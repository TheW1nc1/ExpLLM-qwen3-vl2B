# 方案一可行性分析（严密版）

**方案一**：ShareGPT + accuracy_focused + 增强数据 + 高精度 YAML（dataset: rafdb_emo_aug）

---

## 一、流程与依赖

| 步骤 | 动作 | 前置条件 | 产出 |
|------|------|----------|------|
| 1 | `convert_rafdb_to_llamafactory.py` | 存在 `data_list/anno_json/rafdb_emo_au_train_mini_description.json`，RAF-DB 可访问 | `data_list/llamafactory/rafdb_train_emo.json` |
| 2 | `augment_rafdb_for_llamafactory.py` | 步骤 1 产出存在；RAF-DB 在项目根下（或 media_dir 可指向实际路径） | `rafdb_train_emo_aug.json` + 增强图 + `data/dataset_info.json` 含 rafdb_emo_aug |
| 3 | 训练前路径就绪 | LLaMA-Factory 读 `dataset_dir` + `file_name` → 实际文件存在 | 能加载 rafdb_emo_aug |
| 4 | `llamafactory-cli train` 用 `llamafactory_config_qwen3vl2b_accuracy.yaml`，YAML 内 `dataset: rafdb_emo_aug` | 同上 + data/dataset_info.json 含 rafdb_emo_aug | 训练跑通 |
| 5 | 评估 `valid_qwen3vl_llamafactory.py` | 默认 accuracy_focused、强标签提取、do_sample=False | 与训练指令一致，指标可比 |

---

## 二、断点与必要条件

### 2.1 数据路径（LLaMA-Factory 实际读哪里）

- **规则**：`dataset_dir` 默认 `./data`，`file_name` 相对其解析 → 完整路径 = `data` + `file_name`。
- **当前**：`file_name` = `data_list/llamafactory/rafdb_train_emo_aug.json` → 实际需存在 **`data/data_list/llamafactory/rafdb_train_emo_aug.json`**。
- **原 augment 脚本**：写到 `data_list/llamafactory/rafdb_train_emo_aug.json`（项目根下），**与 LLaMA-Factory 预期路径不一致**，会导致“找不到数据集”。
- **结论**：必须让增强后的 JSON 出现在 `data/data_list/llamafactory/` 下（或改 dataset_dir/file_name 且与脚本写入路径一致）。推荐：augment 脚本直接写入 `data/data_list/llamafactory/rafdb_train_emo_aug.json`。

### 2.2 图片路径（media_dir）

- YAML 中 `media_dir: RAF-DB` → 相对当前工作目录，即 `项目根/RAF-DB`。
- JSON 里图片路径为相对 media_dir：`basic/Image/aligned/xxx.jpg`、`basic/Image/aligned/aug/xxx_aug.jpg`。
- **必要条件**：训练时 cwd = 项目根，且 **RAF-DB 目录在项目根下**（Colab 上即 `PROJECT_ROOT/RAF-DB`）。若 RAF-DB 在别处，需在 YAML 中把 `media_dir` 改为绝对路径或相对项目根的可用路径。

### 2.3 convert 的输入

- `__main__` 默认读 `data_list/anno_json/rafdb_emo_au_train_mini_description.json`（项目内已存在）。
- 若你用别的训练 JSON（名字/路径不同），需改脚本或传参，否则步骤 1 会报“文件不存在”。

### 2.4 augment 的 media_dir

- 脚本里 `media_dir = project_root / "RAF-DB"` 写死。
- **必要条件**：RAF-DB 在项目根下；否则需改脚本支持环境变量或参数指定 RAF-DB 根目录。

### 2.5 dataset_info 覆盖

- augment 会写 **`data/dataset_info.json`**，且合并 `scripts/llamafactory_dataset_info.json` 后写入。
- 若你已在 `data/dataset_info.json` 里配置了其他数据集，augment 会**保留**原有键并新增 `rafdb_emo_aug`（脚本逻辑是读入 base_info 再添加 rafdb_emo_aug）。若脚本是“整体覆盖”写回，则会丢失已有项——需确认当前实现是“合并”而非“覆盖”。

### 2.6 推理与训练一致性

- 训练指令（accuracy_focused）：  
  `"Answer with exactly one word from: Neutral, Happiness, Sadness, Surprise, Fear, Disgust, Anger."`（7 类，无 Contempt，与 convert_rafdb_to_llamafactory 一致）
- 评估脚本默认使用同一段提示，且强标签提取、do_sample=False → **一致，可行**。

---

## 三、结论与必须修复项

- **逻辑上方案一可行**：数据格式、dataset_info 结构、YAML 与评估脚本均对齐；唯一易错点是**数据集文件实际路径**与 LLaMA-Factory 的 `dataset_dir`+`file_name` 解析一致。
- **必须满足**：
  1. **路径**：增强后的 JSON 出现在 `data/data_list/llamafactory/rafdb_train_emo_aug.json`（或同步修改 dataset_dir/file_name 与脚本输出一致）。
  2. **RAF-DB**：训练与 augment 时都能访问同一 RAF-DB 根目录（项目根下或通过可配置路径）。
  3. **convert 输入**：使用的训练源 JSON 存在且路径正确。
  4. **dataset_info**：训练前 `data/dataset_info.json` 中存在 `rafdb_emo_aug`，且 `file_name` 与上述路径规则一致。

**已做**：augment 脚本已改为将 `rafdb_train_emo_aug.json` 写入 `data/data_list/llamafactory/rafdb_train_emo_aug.json`，`file_name` 仍为 `data_list/llamafactory/rafdb_train_emo_aug.json`，LLaMA-Factory 默认 `dataset_dir=data` 即可读到。RAF-DB 不在项目根时可通过环境变量 `RAF_DB_ROOT` 指定。

**补充**：若先用 `rafdb_emo`（未增强）训练，需将 `data_list/llamafactory/rafdb_train_emo.json` 复制到 `data/data_list/llamafactory/`，否则 LLaMA-Factory 会找不到；方案一用 `rafdb_emo_aug` 时由 augment 直接写入 `data/...`，无需再拷。
